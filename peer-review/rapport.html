Question 1

L’une des premières observations issues de notre jeu de données concerne les colonnes Windows, Linux et Mac, qui correspondent aux trois systèmes d’exploitation les plus répandus dans le monde.

Cela soulève naturellement la question suivante :

Parmi ces trois systèmes d’exploitation, lequel prend en charge le plus grand nombre de jeux ?

Question 3

Une autre observation intéressante de notre jeu de données concerne les colonnes price et estimated_owners, qui correspondent respectivement au prix d’un jeu et à son nombre estimé de propriétaires.

Cela soulève la question suivante :

Est-ce que le prix influence les ventes ?

# On ne garde que les jeux qui ont une estimation d'owners
data <- jeux[jeux$estimated_owners != "0 - 0", ]

# On crée une nouvelle variable catégorielle à partir de la variable estimated_owners, en ordonnant les catégories de manière croissante
owner_levels <- unique(data$estimated_owners)
owner_order <- owner_levels[order(sapply(strsplit(owner_levels, " - "), function(x) as.numeric(x[1])))]
data$owners_cat <- factor(data$estimated_owners, levels = owner_order)

# On crée une fonction pour formater les catégories d'estimation d'owners en millions ou milliers, pour une meilleure lisibilité sur le graphique
format_millions <- function(x) {
  sapply(strsplit(x, " - "), function(parts) {
    parts <- as.numeric(parts) # On convertit les parties en numérique pour pouvoir les formater
    fmt <- function(n) {
      if (n >= 1e6) paste0(n/1e6, "M") # Si le nombre est supérieur ou égal à un million, on le divise par un million et on ajoute "M"
      else if (n >= 1e3) paste0(n/1e3, "K") # Si le nombre est supérieur ou égal à mille, on le divise par mille et on ajoute "K"
      else n # Sinon, on laisse le nombre tel quel
    }
    paste0(fmt(parts[1]), " - ", fmt(parts[2])) # On reformate les deux parties de la catégorie d'estimation d'owners et on les concatène avec " - " entre les deux
  })
}

# On calcule le prix moyen et le pourcentage de jeux gratuits pour chaque catégorie d'estimation d'owners
means <- aggregate(price ~ owners_cat, data = data, FUN = mean)
pct   <- aggregate(price ~ owners_cat, data = data, FUN = function(x) mean(x == 0) * 100)
means$pct_gratuit <- pct$price
means$label <- factor(format_millions(as.character(means$owners_cat)), levels = format_millions(as.character(means$owners_cat)))
means$x <- as.numeric(means$label)

# On ajuste une courbe de tendance pour le prix moyen en fonction de l'estimation d'owners, et on la met à l'échelle pour qu'elle soit visible sur le même graphique que les pourcentages de jeux gratuits
scale_factor <- max(means$price) / 100
means$smooth <- pmax(predict(loess(price ~ x, data = means)), 0)

# Visualisation combinée du prix moyen et du pourcentage de jeux gratuits par catégorie d'estimation d'owners
ggplot(means, aes(x = x)) +
  geom_bar(aes(y = pct_gratuit * scale_factor), stat = "identity",
           fill = "#e15759", alpha = 0.35, width = 0.7) + # Barres rouges pour le pourcentage de jeux gratuits
  geom_area(aes(y = smooth), fill = "#4e79a7", alpha = 0.1) + # Zone bleue pour la courbe de tendance du prix moyen
  geom_line(aes(y = smooth), color = "#4e79a7", linewidth = 1.4) + # Ligne bleue pour le prix moyen
  scale_x_continuous(breaks = means$x, labels = means$label) +
  scale_y_continuous(
    name = "Prix moyen ($)",
    sec.axis = sec_axis(~ . / scale_factor, name = "% de jeux gratuits",
                        labels = function(x) paste0(x, "%")) # Axe secondaire pour le pourcentage de jeux gratuits, avec des labels en pourcentage
  ) +

  # On ajoute un titre et un sous-titre pour expliquer les éléments du graphique, ainsi que des thèmes pour améliorer la lisibilité
  labs(
    title = "Est-ce que le prix influence les ventes ?",
    subtitle = "Ligne bleue = prix moyen | Barres rouges = % de jeux gratuits",
    x = "Nombre de propriétaires estimés"
  ) +
  theme_minimal(base_size = 13) +
  theme(
    plot.title = element_text(face = "bold", size = 15),
    plot.subtitle = element_text(color = "grey40", size = 10),
    axis.text.x = element_text(angle = 45, hjust = 1),
    axis.title.y.left = element_text(color = "#4e79a7", face = "bold"),
    axis.title.y.right = element_text(color = "#e15759", face = "bold"),
    panel.grid.minor = element_blank()
  )

Le prix moyen des jeux augmente avec le nombre de propriétaires jusqu’à un certain seuil, puis diminue fortement pour les catégories les plus populaires. On remarque également que la part de jeux gratuits est plus élevée dans les catégories les plus populaires.

On peut donc considérer que le prix influence les ventes, mais les jeux les plus populaires sont souvent gratuits ou peu chers, ce qui montre que l’accessibilité joue un rôle clé lors de l’achat d’un jeu.

Question 4

En parcourant les différentes variables du jeu de données, la mise en relation des colonnes price désignant le prix d’un jeu et pct_pos_total correspondant à la note attribuée au jeu semblait intéressante. En effet, on s’attendrait à ce que plus on mette d’argent dans un jeu, meilleur il soit.

D’où la question suivante :

Est-ce que le prix influence la qualité ?

Tout d’abord, pour répondre à la question, nous avons exploré le jeu de données pour voir s’il était cohérent de représenter dans son entièreté un graphique mettant en relation prix et avis positifs des jeux.

# On ne conserve que les jeux qui ont une note

# Histogramme de la répartition du nombre de jeux en fonction du prix 
ggplot(reviews_prix, mapping = aes(x = price)) + 
  geom_histogram() +
  labs(title = "Répartition du nombre de jeux en fonction du prix", x = "Prix ($)", y = "Nombre de jeux")

Après exploration des données, on se rend compte qu’il y a très peu de jeux dont le prix est supérieur à 70 dollars, ainsi on décide d’écarter ces jeux pour avoir une meilleure visibilité de la tranche la plus représentée (jeux allant de la gratuité à 70 dollars).

# On ne garde que les jeux possédant une note et dont le prix est inférieur à 70 dollars.
reviews_prix <- jeux %>% filter(pct_pos_total > 0, price < 70)

# Graphique présentant la distribution des notes données aux jeux par tranche de prix. (Mettre en relation avec un nuage de points les avis positifs avec les prix rendait le tout illisible.)

ggplot(reviews_prix, mapping = aes(x = cut(price, breaks = 14), y = pct_pos_total)) + # On sépare de manière à avoir des tranches de prix allant de 5 par 5
  geom_boxplot() +
  labs(title = "Distribution des notes données aux jeux par tranche de prix", x = "Tranches de prix ($)", y = "Avis positifs (%)") +
  theme(axis.text.x = element_text(angle = 45, hjust = 1)) # Les intervalles sont illisibles sinon

D’après le graphique pour chaque tranche de prix, le pourcentage d’avis positif reste à priori le même. Ainsi, on ne peut pas vraiment dire que le prix influence la qualité d’un jeu. Cependant, est-il réellement intéressant de voir si le prix influence la qualité sans même savoir le nombre de notes qui ont été attribuées dans chaque tranche ?
